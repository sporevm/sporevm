# SporeVM v0.2.0

Date: 2026-06-21

v0.2.0 is the second pre-1.0 SporeVM baseline. It keeps the aarch64
HVF/KVM scope from v0.1.0 and adds stronger machine introspection, evented run
output, rootfs CAS groundwork, and the Buildkite-hosted release asset flow.

## Scope

- Normalized machine and host JSON output for backend-neutral inspection.
- Run event streams and a bundle inspection contract for tool-facing workflows.
- Human-readable `spore ls` output by default.
- Cached and chunked rootfs CAS groundwork, including repeatable chunk-size
  experiment controls and A1 benchmark evidence.
- System rootfs cache cleanup.
- Default managed run kernel updated to `v0.5.1`.
- Tag-driven Buildkite release automation that builds Linux ARM64 and macOS
  ARM64 archives on their native workers, then publishes GitHub prerelease
  assets from a hosted fan-in job.

## Release Gate

Cut this release only from `main` after:

```console
mise run check
mise run validate:release-a1-kvm -- \
  --region REGION \
  --source-instance SOURCE_INSTANCE \
  --dest-instance DEST_INSTANCE \
  --bucket BUCKET \
  --source-peer-ip SOURCE_PRIVATE_IP
```

The A1/KVM gate covers direct-S3 diskless bundle pull, HTTP-peer rootfs bundle
pull, destination cache reuse, rootfs cache reuse, corrupt bundle/rootfs
rejection, KVM networking smokes, and KVM resume on the destination host.

## Known Limits

- Spore manifests remain pre-1.0 and may break between tags.
- Cross-backend restore remains diagnostic, not the release-critical product
  path.
- Chunked rootfs storage is still groundwork; the product resume path still
  verifies the current rootfs artifact contract.
- Disk-backed named lifecycle suspend/resume remains follow-up work.
