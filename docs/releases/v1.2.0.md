# SporeVM v1.2.0

Date: 2026-06-26

v1.2.0 turns the `libspore` embedding surface into a shipped Zig and C
interface, stabilizes the local named lifecycle path, and tightens rootfs and
resume behavior for image-created spores.

## Highlights

- Adds the `libspore` C ABI with `spore.h`, static and shared libraries,
  `pkg-config` metadata, C smoke coverage, and separate Linux ARM64 and macOS
  ARM64 `libspore` release archives.
- Expands the Zig `libspore` product API with managed fresh runs,
  run-from-spore, typed lifecycle events, host/cache inspection, and bundle
  inspection helpers.
- Stabilizes named lifecycle commands for local `create`, `exec`, `suspend`,
  `resume --name`, `ls`, and `rm` on supported HVF and KVM backends, including
  JSON output and memory reporting in `spore ls`.
- Adds `spore run --pull=missing|always|never` for image references, so callers
  can choose cached image-ref reuse, forced refresh, or fail-closed cache-only
  execution.
- Hardens local object-store reads for dirty RAM chunks and sealed disk layers
  by verifying existing bytes instead of trusting cache path collisions.
- Improves warm resume and fan-out behavior by flushing KVM exec-probe vsock RX,
  waking the guest agent on child exit, and injecting fresh generation identity
  during resume attach.
- Retries benign KVM `KVM_RUN` interrupts instead of failing unrelated guest
  runs when a handled host signal interrupts the vCPU loop.
- Clarifies the writable-rootfs contract: image-created captures preserve rootfs
  writes as sealed disk layers over the immutable image base, while explicit
  `--rootfs PATH` remains a local read-only escape hatch.

## Release Gate

Cut this release only from `main` after:

```console
mise run check
mise run release:snapshot
mise run validate:release-a1-kvm -- \
  --region REGION \
  --source-instance SOURCE_INSTANCE \
  --dest-instance DEST_INSTANCE \
  --bucket BUCKET \
  --source-peer-ip SOURCE_PRIVATE_IP
```

The A1/KVM gate covers direct-S3 diskless bundle pull, direct-S3 and HTTP-peer
chunked-rootfs bundle pulls, destination cache reuse, rootfs CAS cache reuse,
corrupt bundle/rootfs rejection, KVM networking smokes, and KVM resume on the
destination host.

## Known Limits

- Cross-backend restore remains diagnostic, not the release-critical product
  path.
- Named lifecycle is stable only for the local command scope listed above;
  broader lifecycle orchestration remains outside this release.
- Explicit `spore create --rootfs PATH` lifecycle checkpoints still fail closed
  because they do not carry portable immutable-rootfs identity.
- General block-device state remains out of scope. Rootfs-bound writable state is
  represented as sealed disk layers.
