# SporeVM v1.3.0

Date: 2026-06-28

v1.3.0 rounds out the named lifecycle and embedding surfaces: named VMs gain
networking, live fork, annotation metadata, and broader `libspore` coverage,
while CI now publishes benchmark data from the dedicated benchmark pipeline.

## Highlights

- Adds SporeVM-managed networking to named lifecycle VMs, including persisted
  network policy on resume, guest-local bound services, and network audit
  events.
- Adds diskless named live fork and a configurable resume timeout for lifecycle
  callers.
- Expands the Zig, C, and Go `libspore` surfaces for named lifecycle, pull,
  networking, rootfs, and system operations. The C ABI is now version 8, and
  the Go binding requires ABI v8.
- Adds optional manifest `annotations` for embedders that need to carry
  opaque, namespaced metadata such as policy or provenance. SporeVM validates
  UTF-8 keys and values, caps annotations at 64 KiB, preserves them through
  capture, restore, named snapshots, pack, unpack, and pull, and exposes keys
  through `spore inspect`.
- Preserves rootfs identity for lifecycle checkpoints and improves resumed-run
  latency and benchmark phase visibility.
- Reports lifecycle memory policy, configured bytes, chunk counts, and local
  backing-file size information in `spore ls` so named VM memory behavior is
  visible without inspecting manifests by hand.
- Extends image pull policy to named lifecycle create, so mutable image refs can
  use `missing`, `always`, or `never` cache behavior there too.
- Lets `spore run` accept shell commands without `--`, while keeping
  `-- <argv...>` for exact argv.
- Publishes benchmark site data, trend metrics, raw samples, and split cold KVM
  TTI timings from the dedicated benchmark pipeline.
- Uses managed SporeVM kernel release assets for the default run path.
- Updates install, release, and Go module metadata for the `sporevm/sporevm`
  repository home.

## Release Gate

Cut this release only from `main` after:

```console
mise run check
mise run release:snapshot
mise run validate:release-a1-kvm -- \
  --region REGION \
  --source-instance SOURCE_INSTANCE \
  --dest-instance DEST_INSTANCE \
  --bucket BUCKET \
  --source-peer-ip SOURCE_PRIVATE_IP
```

The A1/KVM gate covers direct-S3 diskless bundle pull, direct-S3 and HTTP-peer
chunked-rootfs bundle pulls, destination cache reuse, rootfs CAS cache reuse,
corrupt bundle/rootfs rejection, KVM networking smokes, and KVM resume on the
destination host.

## Known Limits

- Annotation values are intentionally opaque to SporeVM. Use namespaced keys
  such as `dev.buildkite.cleanroom.policy_hash`; SporeVM stores and preserves
  them but does not interpret their meaning.
- Cross-backend restore remains diagnostic, not the release-critical product
  path.
- Named live fork is diskless; rootfs-backed fork remains outside this release.
