# SporeVM v1.3.0

Date: 2026-06-28

v1.3.0 adds opaque manifest annotations for embedders that need to carry
policy, provenance, or other namespaced metadata through SporeVM without
changing restore behavior.

## Highlights

- Adds an optional `annotations` string map to spore manifests. SporeVM
  validates UTF-8 keys and values, caps the serialized annotation object at
  64 KiB, preserves annotations through capture, restore, named lifecycle
  snapshots, pack, unpack, and pull, and exposes annotation keys through
  `spore inspect`.
- Extends the Zig `libspore` named lifecycle APIs with annotation options on
  `CreateNamedOptions` and `SnapshotNamedOptions`.
- Bumps the C ABI to version 8 and adds `SporeAnnotation` pairs for named
  create and snapshot options. The Go binding now requires C ABI v8 and wraps
  named create/snapshot with `map[string]string` annotations.

## Release Gate

Cut this release only from `main` after:

```console
mise run check
mise run release:snapshot
mise run validate:release-a1-kvm -- \
  --region REGION \
  --source-instance SOURCE_INSTANCE \
  --dest-instance DEST_INSTANCE \
  --bucket BUCKET \
  --source-peer-ip SOURCE_PRIVATE_IP
```

The A1/KVM gate covers direct-S3 diskless bundle pull, direct-S3 and HTTP-peer
chunked-rootfs bundle pulls, destination cache reuse, rootfs CAS cache reuse,
corrupt bundle/rootfs rejection, KVM networking smokes, and KVM resume on the
destination host.

## Known Limits

- Annotation values are intentionally opaque to SporeVM. Use namespaced keys
  such as `dev.buildkite.cleanroom.policy_hash`; SporeVM stores and preserves
  them but does not interpret their meaning.
- Cross-backend restore remains diagnostic, not the release-critical product
  path.
