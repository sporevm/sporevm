# SporeVM v1.1.0

Date: 2026-06-24

v1.1.0 turns the post-1.0 CLI implementation into a shared product API for Zig
embedders, removes the lifecycle monitor feature gate, and tightens the
fan-out identity path.

## Highlights

- Adds the public `libspore` Zig module. It exposes typed product operations and
  result contracts for host inspection, raw run, managed fresh run setup,
  run-from-spore, resume, fork, bundle inspect/pack/pull/push/unpack, and
  failure classification while keeping backend, device, storage, daemon, and CLI
  internals out of the public module.
- Routes CLI behavior through the same product APIs that embedders import, so
  CLI and library callers share result/event contracts.
- Removes the `SPOREVM_EXPERIMENTAL_MONITOR` dispatch gate. Named lifecycle
  monitor commands are available by default on supported HVF and KVM backends,
  with monitor failure-mode smoke coverage.
- Embeds the minimal exec initrd in release binaries, so the default exec bridge
  no longer needs a separate initrd asset beside the packaged `spore` binary.
- Fixes resumed fan-out identity delivery. Resumed children receive refreshed
  generation metadata through the guest attach path, keeping
  `/run/sporevm/generation.json`, `/run/sporevm/env`, hostname, and
  `SPORE_*` helper values aligned with the child manifest.

## Release Gate

Cut this release only from `main` after:

```console
mise run check
mise run release:snapshot
mise run validate:release-a1-kvm -- \
  --region REGION \
  --source-instance SOURCE_INSTANCE \
  --dest-instance DEST_INSTANCE \
  --bucket BUCKET \
  --source-peer-ip SOURCE_PRIVATE_IP
```

The A1/KVM gate covers direct-S3 diskless bundle pull, direct-S3 and HTTP-peer
chunked-rootfs bundle pulls, destination cache reuse, rootfs CAS cache reuse,
corrupt bundle/rootfs rejection, KVM networking smokes, and KVM resume on the
destination host.

## Known Limits

- Cross-backend restore remains diagnostic, not the release-critical product
  path.
- Named lifecycle monitor commands are available by default on supported
  backends, but the broader lifecycle surface remains experimental.
- General block-device state remains out of scope. Rootfs-bound writable state is
  represented as sealed disk layers.
