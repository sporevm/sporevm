# SporeVM v1.4.0

Date: 2026-06-30

v1.4.0 turns the post-1.3 runtime work into a release: multi-vCPU runs and
capture/resume are supported on the aarch64 HVF/KVM paths, command UX is less
fussy for humans, and automatic-memory lifecycle behavior has repeatable smoke
coverage.

## Highlights

- Simplifies CLI command forms across `run`, `create`, and `exec`: shell
  commands are accepted by default, while `-- <argv...>` keeps exact argv.
- Adds commandless `spore run --from SPORE` attach, `spore ps` as an alias for
  `spore ls`, terse lifecycle status output, and detached startup commands for
  `spore create`.
- Enables bounded fixed-RAM multi-vCPU runs on HVF and KVM, including manifest
  v1 capture/resume for KVM and same-backend HVF.
- Adds runtime smoke coverage for guest-visible CPU count, multi-vCPU capture,
  `run --from`, `resume`, and lifecycle distribution.
- Adds named lifecycle auto-memory smoke coverage that verifies configured RAM,
  nonzero chunks, and sparse local backing after suspend.
- Adds fs-verity local backing proofs and reports the memory restore source so
  local backing fast paths are easier to reason about.
- Drops fork child local backing metadata when the child proof cannot be
  written, so forked child manifests only advertise usable local backing.
- Moves SporeVM CI jobs onto dedicated SporeVM queues and keeps the triggered
  benchmark image pinned.

## Release Gate

Cut this release only from `main` after:

```console
mise run check
mise run release:snapshot
mise run validate:release-a1-kvm -- \
  --region REGION \
  --source-instance SOURCE_INSTANCE \
  --dest-instance DEST_INSTANCE \
  --bucket BUCKET \
  --source-peer-ip SOURCE_PRIVATE_IP
```

The A1/KVM gate covers direct-S3 diskless bundle pull, direct-S3 and HTTP-peer
chunked-rootfs bundle pulls, destination cache reuse, rootfs CAS cache reuse,
corrupt bundle/rootfs rejection, KVM networking smokes, and KVM resume on the
destination host.

## Known Limits

- Cross-backend restore remains diagnostic, not the release-critical product
  path.
- Multi-vCPU live fork and transient virtio-mem capture/resume remain
  fail-closed.
- Cleanroom-focused Go lifecycle/networking/readback additions are not part of
  this release.
