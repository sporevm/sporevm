# SporeVM v1.0.0

Date: 2026-06-23

v1.0.0 is the first stable SporeVM release for the default aarch64
run/capture/resume/fork/fan-out surface.

## Scope

- One-shot `spore run`, signal capture, `spore resume`, `spore run --from`,
  `spore fork`, and `spore fanout`.
- Immutable OCI rootfs execution with manifest-attached chunked rootfs storage,
  rootfs CAS reuse, writable rootfs disk layers, and bundle pack/unpack/push/pull
  paths for selected children.
- Managed run kernel `v0.5.2`, ReleaseSafe shipping builds, verified kernel cache
  reuse, and verified rootfs/chunk/disk-layer materialization.
- Spore-managed guest networking for DNS, HTTP/HTTPS, persisted egress policy,
  and hard-floor egress denial.
- Named lifecycle `create`, `exec`, `suspend`, `resume --name`, `ls`, and `rm`
  on supported backends, with macOS sandbox and Linux seccomp jailing for denied
  child process execution.

## Release Gate

Cut this release only from `main` after:

```console
mise run check
mise run release:snapshot
mise run validate:release-a1-kvm -- \
  --region REGION \
  --source-instance SOURCE_INSTANCE \
  --dest-instance DEST_INSTANCE \
  --bucket BUCKET \
  --source-peer-ip SOURCE_PRIVATE_IP
```

The A1/KVM gate covers direct-S3 diskless bundle pull, direct-S3 and HTTP-peer
chunked-rootfs bundle pulls, destination cache reuse, rootfs CAS cache reuse,
corrupt bundle/rootfs rejection, KVM networking smokes, and KVM resume on the
destination host.

## Known Limits

- Cross-backend restore remains diagnostic, not the release-critical product
  path.
- Explicit `spore create --rootfs PATH` lifecycle VMs cannot be suspended into
  portable disk-backed spores; use `--image` when checkpointing rootfs state.
- General block-device state remains out of scope. Rootfs-bound writable state is
  represented as sealed disk layers.
